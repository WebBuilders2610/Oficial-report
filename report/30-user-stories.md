# Chapter III: Requirements Specification

## 3.1. User Stories

En esta sección se presentan las User Stories que definen los requerimientos funcionales del sistema propuesto, orientado al monitoreo inteligente de la salud neonatal. Estas historias de usuario constituyen la base del enfoque ágil adoptado, permitiendo describir de manera clara y centrada en el usuario las funcionalidades necesarias para resolver el problema crítico identificado: la falta de visibilidad de señales de alerta temprana en recién nacidos debido a la ausencia de registros objetivos y estructurados.

El sistema plantea una solución basada en una plataforma digital con una base de datos centralizada que permite registrar, analizar y monitorear parámetros clave del bebé (como temperatura, peso, alimentación y coloración). A través de un Tablero de Salud Inteligente, se validan los datos frente a estándares médicos y se generan alertas automáticas ante posibles anomalías, contribuyendo a una toma de decisiones más oportuna y fundamentada.

Las User Stories se han organizado utilizando una jerarquía de dos niveles que facilita la comprensión, trazabilidad y gestión de los requerimientos:

- **Epics (Épicas):** Representan conjuntos amplios de funcionalidades alineadas a objetivos clave del sistema, como el monitoreo de datos neonatales, la generación de alertas inteligentes, el análisis de información clínica y la gestión de usuarios. Cada épica agrupa historias relacionadas que en conjunto aportan valor significativo a la solución.

- **User Stories (Historias de Usuario):** Son descripciones específicas de funcionalidades desde la perspectiva de los distintos tipos de usuario (padres, neonatólogos, visitantes y desarrolladores. Está acompañada de criterios de aceptación que definen las condiciones necesarias para su validación. Estas siguen el siguiente formato: "Como {tipo de usuario}, quiero {funcionalidad} para {beneficio}". 

#### Segmentos Objetivo:

Las User Stories han sido diseñadas considerando los principales actores del sistema:

- **Padres de recién nacidos:** Enfocados en registrar datos diarios, recibir alertas oportunas, visualizar reportes y llevar un control continuo del estado de salud del bebé desde el hogar.

- **Neonatólogos y personal de salud:** Orientados a consultar historiales completos, generar reportes clínicos, monitorear múltiples pacientes y recibir alertas críticas para priorizar la atención médica.

#### Criterios de Aceptación y Validación:

Cada User Story incluye criterios de aceptación definidos en formato Gherkin (Given-When-Then), lo cual permite:

- Establecer comportamientos esperados de forma clara y verificable.
- Servir como base para la creación de pruebas funcionales y automatizadas.
- Validar objetivamente la correcta implementación de cada funcionalidad.
- Facilitar la comunicación entre stakeholders, desarrolladores y equipo de testing.

Esta estructura garantiza que el sistema no solo cubra las necesidades funcionales, sino que también responda de manera efectiva al problema central: mejorar la detección temprana de riesgos en recién nacidos mediante el uso de datos confiables, alertas inteligentes y herramientas de análisis clínico.
